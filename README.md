# Unity RTS Game Project
## 🎯 项目目标

本项目旨在基于 Unity 引擎 开发一款即时战略（RTS）类型的游戏，探索 单位控制、AI 行为、巡逻机制 等核心玩法，并通过逐步迭代的方式完善战斗与策略系统。

## 📖 项目内容

- 实现基本的 单位移动与控制

- 引入 巡逻与警戒机制，单位能够自动发现并响应敌人

- 搭建 战斗与武器系统 的框架

- 探索 性能优化 与 资源管理

- 借鉴经典 RTS（如《红警3》）的机制，尝试兼容多种巡逻与战斗算法

- 规划未来接入 智能 AI，让游戏对战更具挑战性和互动性

- 与搜打撤玩法结合

## 游戏设计方案
### 整体游戏循环

基地整备 -> 准备下图单位 -> 进入地图 -> 探索地图，收集物资，战斗 -> 单位部分撤离 -> 继续探索 -> 所有单位撤离 -> 回到基地 -> 家园建设 -> 基地整备

**然而这会引出一系列问题**
- 下图的单位应该有多少？太多了会提高操作难度，太少了就没意思了
- RTS必备的战地基地建设该如何进行？
- AI敌人如何与世界交互？是模拟玩家进入地图探索，还是在固定区域巡逻？或者是巡猎追击玩家
- 每个单位都要设置物品栏吗？是否太麻烦，但如果不设置该如何适配部分单位撤离这个机制？
- 如果在有单位没撤离时就离开当局游戏，这些单位该如何处理？
- 家园建设所用的资源该怎么和地图内搜到的物资产生联系？
- 如果将所有单位视为资产，会不会导致玩家战斗的过于保守？

#### 关于战斗系统
PVERTS游戏的一大乐趣点就在于组织部队推进，这一点在红色警戒心灵终结中感受颇为深刻。然而搜打撤玩法注定了单位数量不能太多，大规模部队推进似乎并不是一个很合适的玩法。但是如果全程小队作战会让游戏过程十分紧张，而且容错率很低。可以给玩家提供类似心灵终结中斯大林之拳的可展开生产单位作为战地生产的载体，来补充部队，但这种单位应该比较稀有，我们可以设定装甲类单位在战场时极难补充的，而普通的补兵单位可以通过类似空降伞兵的支援技能作为战地补充单位的方式，同时降低基础单位的成本，让其成为可以被消耗的单位。不过不能一开局就直接解锁这类技能，应该设定条件，比如必须干掉地图上的雷达、防空系统之后才能进行空降支援。

#### 关于支援技能
支援技能应该作为游戏重要的辅助玩法之一。提供诸如侦查，回复，辅助增伤，临时电力的功能。这里很大程度上参考的红色警戒3和红色警戒心灵终结。同时飞机空袭也可以作为支援技能，而飞机的提供则依赖于家园系统。

#### 关于单位控制
很有必要制作小队单位控制，让不同类型，不同移速的单位聚集在一起移动，甚至可以形成前卫单位和后排单位的区别，同时让他们有能力在遭遇敌人时自动反击，来降低控制难度。

#### 关于单位撤离与物资带回
的确不能设置每个单位的物品栏，这会导致操作难度直接飙升（致命冲击.jpg）。可以设置专门的物资卡车，来存储物资，或者是让基地车可以携带物资，我们把基地车的携带物资的能力看做玩家的背包，而卡车之类的东西可以散布在地图中，作为额外的背包空间。同时，地图中可以提供诸如火车、机场、富尔顿回收系统等方式来带出物资或者作为特定类型单位的撤离方式。

#### 当前的计划
整体开发的量十分巨大，所以应该做一些取舍，目前需要做的是：
- 基地车以及其带有的背包
- 一些基础步兵，装甲单位
- 撤离机制
- 简单的家园机制（用于体现撤离后的内部家园机制和单位整备机制）
- 简单的支援技能
目前战斗系统已经基本完成，所以在这个基础上完成上述内容，就可以算做一个完整的MVP了
